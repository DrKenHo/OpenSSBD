What is SSBD API?

SSBD API is the Application Programming Interface for accessing the internal data of the SSBD database. The API is based on the web development practice of using RESTFul architecture. A RESTful API call is simply an HTTP request to the SSBD server.

Prerequisite

Before you begin, you need to be familiar with BDML file, its terminology and its schema. Please refer to ​BDML version 0.18 Specification for more information.
Although the internal data structure is different from the BDML file format, much of its terminology and the data structure are derived from BDML file format.

Summary of SSBD API

API Request Method Functions Status
/SSBD/api/v1/ GET list all the available APIs
/SSBD/api/v1/bdml/ GET retrieve bdmlUUID depreciated; only support BDML v0.15; see data api instead
/SSBD/api/v1/contact/ GET retrieve the contact details of a BDML file record depreciated; only support BDML v0.15; see data api instead
/SSBD/api/v1/coordsXYZ/ GET retrieve the coordinates and other details in the BDML file record only applicable to BDML ver 0.15
/SSBD/api/v1/data/ GET retrieve all the meta data of a BDML file record replaces info, summay and contact api
/SSBD/api/v1/info/ GET retrieve the info details of a BDML file record depreciated; only support BDML v0.15; see data api instead
/SSBD/api/v1/meta_data/ GET retrieve meta data of a BDML file record similar to data api
/SSBD/api/v1/scale/ GET retrieve the scales used in the BDML file record only applicable to BDML ver 0.15
/SSBD/api/v1/summary/ GET retrieve the summary details of a BDML file record depreciated; only support BDML v0.15; see data api instead

SSBD API Schema

There are two default formats that the API will return. It can either return the data in XML or JSON formats.

bdml API

Description
Retrieve the unique UUID information for the BDML data. Depreciated; Only supports BDML v0.15; Use data api instead
Fields Description Usage example Usage comment
bdml_ID unique BDML UUID identity number of the BDML data stored in SSBD http://ssbd.qbic.riken.jp/SSBD/api/v1/bdml/?format=xml;bdml_ID__icontains=247 To retrieve bdmlUUID which contains '247' from SSBD
title Title of the BDML data stored in SSBD http://ssbd.qbic.riken.jp/SSBD/api/v1/bdml/?format=xml;title__icontains=coli To retrieve the title which contains the phrase 'coli' from SSBD
id internal unique id for data stored in SSBD http://ssbd.qbic.riken.jp/SSBD/api/v1/bdml/?format=xml;id=1 To retrieve the data with internal id=1 from SSBD

​http://ssbd.qbic.riken.jp/SSBD/api/v1/bdml/schema/?format=xml

data API

Description
Retrieve all the meta data described in the BDML file stored in SSBD. It replaces contact, info and summary API. It also covers BDML versions including 0.15 and 0.18.
Fields Description Usage example Usage comment
bdmlUUID unique BDML UUID of the BDML data stored in SSBD http://ssbd.qbic.riken.jp/SSBD/api/v1/data/?format=xml;bdmlUUID__icontains=2475d To retrieve all the meta data from SSBD for bdmlUUID which contains "245d". Note that the field is changed from previous bdml_ID to bdmlUUID in the new API.
bdml_multipart_type it denotes whether the bdmlUUID data is part of a series. 1 stands for standalone file and 2 stands for file in a series. NA Cannot be searched; retrieval only
external_bdml link to the actual bdml file NA Cannot be searched; retrieval only
external_pdpml link to the actual pdpml file NA Cannot be searched; retrieval only
external_source link to the actual source of the data NA Cannot be searched; retrieval only
localid Internal or local identifier used to identify the BDML data. http://ssbd.qbic.riken.jp/SSBD/api/v1/localid/?format=xml;localid__icontains=B0336 To retrieve summary data from SSBD for localid that contains the phrase "B0336"
release date of the release of the data NA Cannot be searched; retrieval only
schema_ver version of the BDML schema used NA Cannot be searched; retrieval only
status status of the data NA Cannot be searched; retrieval only; only 'available' status is allowed to be retrieved
meta_data other meta-data of the BDML file data on SSBD; Searches require using "meta_data__" as prefix of the fields being search; Also see meta_data API http://ssbd.qbic.riken.jp/SSBD/api/v1/data/?format=xml;meta_data__name__icontains=water To retrieve data given that the "name" field in meta_data API contains the phrase "water"
